Aplikasi Satu Data

Pilar strategis untuk integrasi data, transparansi publik, dan fondasi bagi ekosistem aplikasi digital yang terpadu di Kota Baubau.

SOP Pembuatan Aplikasi

Setiap proses pembuatan aplikasi, dari inisiasi hingga pemeliharaan, harus mengikuti alur standar untuk menjamin kualitas, keamanan, dan integrasi.

1. Perencanaan

Analisis kebutuhan, studi kelayakan, dan identifikasi kebutuhan data yang akan diproduksi atau dikonsumsi.

2. Perancangan

Menentukan arsitektur sistem, desain basis data, alur integrasi API, serta merancang UI/UX dan skema keamanan.

3. Pengembangan

Memilih tech stack, menulis kode sesuai praktik terbaik, dan membangun endpoint API sesuai standar RESTful.

4. Pengujian

Melakukan unit testing, integration testing, security testing (pentest), dan User Acceptance Test (UAT).

5. Deployment

Mengonfigurasi lingkungan server (production, staging) dan melakukan rilis menggunakan metode CI/CD.

6. Pemeliharaan

Memantau kinerja dan uptime, serta merilis perbaikan bug dan patch keamanan secara berkala.

Studi Kasus: Peningkatan PAD

Salah satu implementasi utama dari Aplikasi Satu Data adalah untuk Dinas Pendapatan Daerah (Dispenda). Dengan mengintegrasikan seluruh data penerimaan daerah—mencakup modul pajak (PBJT, PBB-P2, BPHTB) dan retribusi (parkir, sampah, pasar)—ke dalam satu platform, proses koleksi PAD menjadi lebih efisien dan transparan.

Aplikasi ini dapat diakses secara online maupun offline, memungkinkan pemantauan real-time terhadap potensi dan realisasi pendapatan, serta memudahkan wajib pajak dalam memenuhi kewajibannya.

Persyaratan Teknis Wajib untuk Integrasi

Setiap aplikasi baru yang dikembangkan wajib memenuhi standar teknis berikut untuk dapat terintegrasi dengan ekosistem Satu Data Baubau.

Arsitektur & Teknologi

Dianjurkan mengadopsi Microservices & Docker. Aplikasi wajib bersifat Stateless.

Standar API

Wajib RESTful, format JSON, dan dokumentasi menggunakan OpenAPI (Swagger).

Keamanan

Komunikasi wajib HTTPS (TLS 1.2+). Autentikasi menggunakan OAuth 2.0 & JWT.

Manajemen Data

Wajib mengambil data master dari API Satu Data dan menggunakan kode referensi standar.
